{{ icon('gear', size=22) }} LLM 設定 附加功能

LLM AI 加值是附加功能，預設關閉。啟用後共 10 個工具（pdf-fill 視覺校驗、 doc-deident / text-deident 智慧分類、doc-diff / pdf-extract-text / pdf-wordcount / pdf-annotations 內容摘要、pdf-ocr 文字校正、submission-check 送件檢核、translate-doc 逐句翻譯）可選擇性地把資料送給本機或 LAN 上的 LLM server (OpenAI-compat) 處理。 本機 / LAN 上需有運行中的 LLM backend (Ollama / vLLM / LM Studio / jan.ai 皆可)。

⚠
使用反向代理（nginx / Caddy）必看 — 504 Gateway Timeout 預防
LLM 推理單筆常 5-15 分鐘（翻譯 / 視覺校驗 / 大模型）。若 jt-doc-tools 放在 nginx 後面， nginx 預設 60 秒就會回 504，本頁下方 Timeout 設多少都救不了。
  • nginx 設 proxy_read_timeout 900s; + proxy_send_timeout 900s;（≥ 下方 Timeout 值 + 60s 緩衝）
  • 多層反向代理（例：自架 LLM proxy 在前 + jt-doc-tools 在後，各自有 nginx）：每一層 nginx 都要設，任一層用預設就斷
  • Caddy 對應加 read_timeout 900s / write_timeout 900s
詳見 OPS.md 504 排錯流程

1. 啟用

關閉時，所有 LLM 邏輯不會執行，pdf-fill / doc-deident / translate-doc 等 10 個工具的 AI 加值會自動隱藏，純規則處理不受影響。

2. 連線資訊

⚠ 隱私建議：強烈建議接 地端自架 LLM Server（如 Ollama / vLLM / LM Studio），讓檔案內容只在公司內網流動。 若指向 OpenAI / Claude / Gemini 等雲端 API，所有送 LLM 校驗 / 翻譯 / 段落重排的原始文件內容都會離開公司網路， 可能違反個資法、營業秘密或客戶 NDA。地端 LLM 才能符合本平台「資料留在自己手中」的設計初衷。
雲端 OpenAI-compat 服務才需要；本地 Ollama / vLLM 留空。
秒（單次 HTTP call 上限；翻譯 / 視覺校驗 / 大模型推理常 5-15 分鐘）
⚠ 反向代理（nginx / Caddy）對應設定一定要一起改：
若 jt-doc-tools 在 nginx 後面，nginx 的 proxy_read_timeout + proxy_send_timeout 必須 ≥ 此 Timeout 值 + 60s 緩衝（建議 900s）。
若有「LLM proxy 也在另一台 nginx 後面」這類多層架構，每一層 nginx 都要設 — 任何一層用預設 60s 就會 504。 詳見 OPS.md 504 排錯流程。
